单调队列与单调栈

单调栈与单调队列

一、前置知识回顾

栈和队列是算法竞赛中最基础的线性数据结构，也是单调栈、单调队列的核心基础：

  1. 栈：先进后出的线性结构，仅能在栈顶进行插入、删除操作，核心操作是push(入栈)、pop(出栈)、top(取栈顶)。
  2. 队列：先进先出的线性结构，在队尾插入、队首删除，核心操作是push(入队)、pop(出队)、front(取队首)、back(取队尾)。

二者的暴力解法在处理n≤2e5的大数据范围时会达到O(n²)的时间复杂度，必然超时；而单调栈/队列通过维护元素的单调性，让每个元素仅入队/入栈、出队/出栈各一次，整体时间复杂度严格为O(n)，是处理线性序列区间最值、边界查找问题的核心工具。

二、单调栈核心讲解

2.1 定义与核心原理

单调栈是栈内元素始终保持单调递增/单调递减的特殊栈结构，核心解决「查找元素左侧/右侧第一个比它大/小的元素」类问题，也是所有复杂单调栈应用的基础。

问题类型 栈的单调性选择 核心作用
找右侧第一个更大元素（NGE） 单调递减栈 栈顶始终是当前未找到答案的元素里最小的
找右侧第一个更小元素（NSE） 单调递增栈 栈顶始终是当前未找到答案的元素里最大的
找左侧第一个更大元素（LGE） 单调递减栈 从右往左遍历，逻辑与NGE一致
找左侧第一个更小元素（LSE） 单调递增栈 从右往左遍历，逻辑与NSE一致

2.2 经典例题精讲

例题1：求下一个更大数（模板题）

题目描述：给定长度为n的数列，求每个数字往后看第一个比它大的数字的下标，不存在则输出0。

数据范围1≤n≤2e5。

样例输入：7 2 6 3 1 5 7 4
样例输出：2 6 5 5 6 0 0

核心思路：使用单调递减栈，栈中存储元素下标（而非数值），保证栈内下标对应的数值单调递减。遍历数组时，若当前元素大于栈顶元素对应的数值，则栈顶元素的答案就是当前下标，弹出栈顶；重复此过程直到栈空或栈顶元素更大，再将当前下标入栈。

例题2：柱状图中最大的矩形（经典应用题）

题目描述：给定n列柱子的高度，求柱子能组成的最大矩形面积。

数据范围1≤n≤2e5。

样例输入：5 1 2 5 3 4
样例输出：9

核心思路：对每个柱子，找到左侧第一个比它矮的柱子下标L[i]、右侧第一个比它矮的柱子下标R[i]，则以该柱子为高的矩形宽度为R[i]-L[i]-1，面积为a[i]*(R[i]-L[i]-1)，遍历所有柱子取最大值即可。左右边界的查找通过两次单调递增栈实现。

三、单调队列核心讲解

3.1 定义与核心原理

单调队列是队列内元素始终保持单调递增/单调递减的特殊队列结构，核心解决滑动窗口最值问题，以及「限定区间长度的最值/子段和」类问题，同样保持O(n)的线性时间复杂度。

核心特性：

  1. 队首元素始终是当前窗口的最值（最大值对应递减队列，最小值对应递增队列）；
  2. 需维护窗口的合法性：队首元素超出窗口范围时，从队首弹出；
  3. 维护单调性：新元素入队时，从队尾弹出所有破坏单调性的元素，再入队。

3.2 经典例题精讲

例题3：滑动窗口最大值（模板题）

题目描述：给定长度为n的数列，输出每个长度为m的连续区间的最大值。

数据范围1≤m≤n≤2e5。

样例输入：7 3 1 5 3 2 2 4 1
样例输出：5 5 3 4 4

核心思路：使用单调递减队列，队列存储元素下标，保证队列内下标对应的数值单调递减。

遍历过程中：

  1. 队尾弹出所有小于等于当前元素的下标，维护单调性；
  2. 当前下标入队；
  3. 弹出队首超出窗口范围的下标；
  4. 遍历到第 m 个元素后，队首就是当前窗口的最大值。

四、单调队列优化DP

4.1 核心原理

当DP状态转移方程满足「在一个滑动窗口内取最值」的形式时，可通过单调队列将DP的时间复杂度从O(nk)优化到O(n)。

适用场景的DP方程特征

设DP数组为dp[i]，常见的可优化方程形式：

  1. 最小值优化：dp[i] = min{ dp[j] + w[i,j] } + c[i]，其中j ∈ [i-k, i-1]，且w[i,j]可分离为仅与i或仅与j相关的项；
  2. 最大值优化：dp[i] = max{ dp[j] + w[i,j] } + c[i]，其中j ∈ [L(i), R(i)]（L(i)、R(i)为i的线性函数）。

优化核心步骤

  1. 拆分转移方程：将方程中与j相关的项和与i相关的项分离；
  2. 确定滑动窗口范围：明确j的取值区间[L, R]；
  3. 维护单调队列：队列存储j的下标，保证队列内dp[j] + w[j]单调递增/递减；
  4. 状态转移：取队首的最值完成dp[i]的计算。

4.2 经典例题精讲

例题4：多重背包（单调队列优化版）

题目描述：有n种物品，第i种物品的体积为v[i]，价值为w[i]，数量为c[i]，背包容量为m，求最大价值。数据范围1≤n≤1e3, 1≤m≤1e5。

暴力DP分析：

  • 状态定义：dp[j]表示容量为j的背包的最大价值；
  • 转移方程：dp[j] = max{ dp[j - k*v[i]] + k*w[i] }，其中0≤k≤c[i]；
  • 时间复杂度：O(n*m*c[i])，当m=1e5时会超时。

单调队列优化思路：
对每个物品i，按j % v[i]分组（余数为r，0≤r<v[i]），每组内的容量可表示为j = r + t*v[i]。
设dp[r + t*v[i]] = f[t]，则转移方程可转化为：
f[t] = max{ f[t - k] + k*w[i] }，其中1≤k≤c[i]，即f[t] = max{ f[s] - s*w[i] } + t*w[i]，s ∈ [t - c[i], t - 1]。
此时max{ f[s] - s*w[i] }是滑动窗口内的最值，可通过单调递减队列维护。

例题5：烽火传递（线性DP+单调队列优化）

题目描述：有n个烽火台，第i个烽火台点燃的代价为w[i]，要求任意连续m个烽火台中至少有一个被点燃，求最小总代价。数据范围1≤n,m≤2e5。

状态定义：dp[i]表示点燃第i个烽火台，且前i个烽火台满足条件的最小代价。
转移方程：dp[i] = min{ dp[j] | j ∈ [i-m, i-1] } + w[i]，最终答案为min{ dp[j] | j ∈ [n-m+1, n] }。

核心思路：转移方程中min{ dp[j] }是长度为m的滑动窗口最小值，用单调递增队列维护即可。

例题6：玩具装箱（斜率优化+单调队列，进阶）

题目描述：有n个玩具，第i个玩具的长度为c[i]，将玩具分成若干段，每段i~j的费用为(j-i + sum(c[i..j]) - L)^2（L为给定常数），求最小总费用。数据范围1≤n≤5e4。

状态定义：dp[i]表示前i个玩具装箱的最小费用。
转移方程：设s[i] = sum(c[1..i])，则dp[i] = min{ dp[j] + (i-j + s[i]-s[j] - L)^2 }（0≤j<i）。

优化思路：
展开方程得：dp[i] = min{ dp[j] + (s[i]+i - (s[j]+j) - L)^2 }。
令a[i] = s[i] + i，b = L + 1，则dp[i] = min{ dp[j] + (a[i] - (a[j]+b))^2 }。
该方程满足斜率优化条件，可通过单调队列维护下凸壳，将转移复杂度从O(n)降至O(1)，整体时间复杂度O(n)。

4.3 单调队列优化DP的核心技巧

  1. 状态分组技巧：多重背包优化中，按j % v[i]分组是关键，将二维的容量问题转化为一维的滑动窗口问题；
  2. 方程拆分原则：必须将转移方程拆分为「仅与j相关」和「仅与i相关」的两部分，才能用单调队列维护最值；
  3. 虚拟初始状态：烽火传递中引入dp[0]=0的虚拟状态，避免处理边界条件，简化队列初始化；
  4. 精度处理：斜率优化中，优先用叉乘代替除法，避免浮点数精度误差。

五、通用编程技巧与避坑指南

5.1 核心编程技巧

  1. 数组模拟栈/队列，优先于STL容器
    竞赛中n≥1e5时，数组模拟的栈/队列比STL的stack/deque速度更快，能有效避免超时；同时数组下标访问更直观，方便调试。

  2. 存储下标，而非数值
    单调栈/队列中永远优先存储元素下标，而非元素本身。通过下标可以随时获取数值，还能计算区间长度、判断窗口范围、定位元素位置。

  3. 严格/非严格单调性的精准控制
    循环条件中的>/>=、</<=直接决定单调性的严格性，需根据题意选择：

    • 元素互不相同：严格/非严格无区别；
    • 元素可重复：求「第一个大于」用>，求「第一个大于等于」用>=。

  4. DP优化的预处理
    单调队列优化DP前，需先预处理前缀和、状态拆分等辅助数组，避免在队列操作中重复计算，提升代码效率。

5.2 高频避坑指南

  1. 数据类型溢出问题
    涉及平方、乘积的DP问题（如玩具装箱），必须用long long类型，避免int溢出导致答案错误。
  2. 窗口范围的边界计算
    滑动窗口的左边界计算易出错，例如j ∈ [i-m, i-1]需确保q[front] >= i-m，而非q[front] > i-m。
  3. 队列初始化的虚拟状态
    DP优化中，若初始状态（如dp[0]）需要参与转移，必须先将其加入队列，否则会遗漏初始值。
  4. 多组测试用例的重置
    处理多组测试时，需重置dp数组、队列的front和rear，避免上一组数据残留。
