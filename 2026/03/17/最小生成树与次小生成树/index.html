最小生成树与次小生成树

最小生成树（MST）与次小生成树（SMST）

前置知识要求

学习前，需熟练掌握以下内容：

  1. 图的基础存储方式（邻接表/链式前向星）

  2. 并查集（路径压缩+按秩合并，时间复杂度近似 $O(\alpha(n))$ ）

  3. 树的DFS/BFS遍历、树上路径的核心性质

  4. 最近公共祖先（LCA）的倍增算法与Tarjan离线算法核心原理

  5. 带权并查集的基础维护方法

第一部分 最小生成树（MST）核心理论与实现

1.1 定义与核心性质

定义

给定无向连通带权图 $G=(V,E)$ ，生成树 $T$ 是包含 $G$ 全部 $n$ 个顶点、边集为 $E$ 的子集、无环的连通子图。最小生成树（Minimum Spanning Tree, MST） 是所有生成树中，边权总和最小的生成树。

核心性质（所有MST算法的理论基石，也是次小生成树的核心依据）

  1. 存在性：若图 $G$ 连通，则MST一定存在；若图中所有边权互不相同，则MST唯一。

  2. 切割性质：将顶点集划分为两个非空子集 $S$ 和 $T$ ，所有连接 $S$ 与 $T$ 的跨集边中，权值最小的边一定属于某一棵MST。

  3. 回路性质：对于图中的任意一个环，环上权值最大的边一定不属于任何MST。

  4. 边权唯一性：任意MST的边权多重集合是唯一的。即无论MST形态如何，不同权值的边的数量固定不变。

1.2 Kruskal算法（稀疏图首选，竞赛最常用）

算法设计思路

基于贪心策略+并查集实现，核心逻辑是「边从小到大选，不形成环就加入」：

  1. 将图中所有边按权值从小到大排序；

  2. 初始化并查集，每个顶点独立成一个连通块；

  3. 按权值从小到大遍历边，若当前边连接的两个顶点属于不同连通块，则将该边加入MST，合并两个连通块；

  4. 当选中 $n-1$ 条边时，MST构建完成；若遍历完所有边仍不足 $n-1$ 条，说明原图不连通，无MST。

时间复杂度

核心开销为边的排序，总时间复杂度 $O(m\log m)$ ，在稀疏图（ $m$ 与 $n$ 同阶）中效率最优，是竞赛中MST的首选算法。

竞赛可用完整模板（含树边标记，为次小生成树做准备）

1.3 Prim算法（稠密图首选）

算法设计思路

基于贪心策略实现，核心逻辑是「点从小到大加，每次选离已选集最近的点」，分为朴素版与堆优化版：

  1. 初始化：任选一个起点（通常为1号点），维护数组dist[]，dist[u]表示顶点 $u$ 到已加入MST的顶点集的最小边权，初始时除起点外所有dist设为无穷大；

  2. 循环 $n$ 次：每次选出未加入MST的、dist最小的顶点 $u$ ，将其加入MST，累加dist[u]到MST总权值；

  3. 松弛更新：用 $u$ 的所有邻边，更新未加入MST的顶点的dist值；

  4. 若循环结束后加入MST的顶点数不足 $n$ ，说明原图不连通。

时间复杂度

  • 朴素版： $O(n^2)$ ，适合稠密图（ $m$ 接近 $n^2$ ），如 $n\le 10^3$ 的完全图；

  • 堆优化版： $O(m\log n)$ ，稀疏图中效率与Kruskal相当，代码复杂度略高。

第二部分 次小生成树（SMST）核心理论

2.1 定义

  • 非严格次小生成树：边权总和 $\ge$ MST边权和，且在所有满足该条件的生成树中权值和最小；

  • 严格次小生成树：边权总和 $>$ MST边权和，且在所有满足该条件的生成树中权值和最小。

竞赛中90%以上的题目考察严格次小生成树，本讲义以严格次小为核心讲解。

2.2 核心定理：单条边替换原理

次小生成树一定可以通过替换MST中的恰好一条边得到。

  • 证明核心：若次小生成树与MST有 $k\ge2$ 条边不同，可通过逐步替换边，将不同边数减少到1，且权值和不增加。因此只需枚举所有「替换一条边」的情况，即可找到次小生成树。

2.3 算法整体框架

  1. 用Kruskal/Prim求出原图的MST，记录MST总权值mst_sum，标记树边与非树边，构建MST的邻接表；

  2. 对于原图中所有非树边 $e=(u,v,w)$ ，将其加入MST会形成唯一的环；

  3. 我们需要删除环上除 $e$ 外的最大权值边：若最大边权 $<w$ ，则替换后权值为mst_sum - max_w + w；若最大边权 $=w$ ，则删除次大权值边，替换后权值为mst_sum - sec_w + w；

  4. 遍历所有非树边，取所有替换后权值的最小值，即为严格次小生成树的总权值。

核心问题：如何快速求MST上 $u$ 到 $v$ 路径的最大/次大边权？

竞赛主流解决方案：倍增法（在线） 与 Tarjan离线法。

第三部分 次小生成树的双算法实现

3.1 倍增法（在线算法，竞赛首选）

算法设计思路

基于倍增LCA实现，核心是预处理树上路径的最大/次大边权，支持在线查询任意两点路径的边权信息：

  1. 预处理倍增数组：

    • up[u][k]：节点 $u$ 向上跳 $2^k$ 步的祖先节点；

    • max1[u][k]：节点 $u$ 到up[u][k]路径上的最大边权；

    • max2[u][k]：节点 $u$ 到up[u][k]路径上的严格次大边权。

  2. DFS初始化：对MST进行DFS，初始化每个节点的父节点up[u][0]、到父节点的边权max1[u][0]，次大边权max2[u][0]设为负无穷。

  3. 倍增数组递推：对于 $k$ 从1到 $\log_2 n$ ，递推更新up、max1、max2数组，核心是合并两段路径的最大/次大边权。

  4. 路径查询：对于任意两点 $u,v$ ，在求LCA的过程中，合并 $u\to LCA$ 和 $v\to LCA$ 两段路径的最大/次大边权，得到完整路径的边权信息。

  5. 枚举非树边计算答案：遍历所有非树边，计算替换后的权值，取最小值即为严格次小生成树的权值。

核心难点：路径最大/次大边权的合并

给定两段路径的最大边权 $a1,b1$ ，次大边权 $a2,b2$ ，合并后的严格最大/次大边权计算逻辑：

  1. 先取合并后的最大值new_max = max(a1, b1)；

  2. 再从所有不等于new_max的边权中取最大值，作为合并后的次大值new_sec；

  3. 候选值包括： $a1,a2,b1,b2$ ，需过滤掉等于new_max的值，再取最大。

时间复杂度

  • 预处理： $O(n\log n)$

  • 单次路径查询： $O(\log n)$

  • 总时间复杂度： $O(m\log n)$ ，可处理 $n\le 1e5$ 的绝大多数竞赛题。

竞赛可用完整模板（严格次小生成树）

3.2 Tarjan离线法（离线算法，效率更优）

算法设计思路

基于Tarjan离线LCA与带权并查集实现，核心是批量处理所有非树边的查询，在一次DFS中完成所有路径的最大/次大边权计算：

  1. 预处理：用Kruskal求MST，构建MST邻接表，收集所有非树边作为查询，每个查询存储 $u,v,w$ ，并将查询分别加入 $u$ 和 $v$ 的查询列表。

  2. 带权并查集设计：除基础的父节点数组外，额外维护两个数组：

    • max1[u]：节点 $u$ 到其并查集根节点的路径上的最大边权；

    • max2[u]：节点 $u$ 到其并查集根节点的路径上的严格次大边权。

  3. Tarjan DFS遍历：

    • 标记当前节点 $u$ 为已访问，遍历 $u$ 的所有树边，递归处理子节点 $v$ ，递归结束后将 $v$ 的并查集父节点设为 $u$ ，并维护max1[v]和max2[v]；

    • 遍历 $u$ 的所有查询，若查询的另一个节点 $v$ 已被访问，则它们的LCA为find(v)，此时合并 $u$ 到根、 $v$ 到根的路径的最大/次大边权，得到 $u$ 到 $v$ 路径的边权信息，计算替换后的权值，更新次小生成树答案。

  4. 路径压缩维护：带权并查集的find函数需在路径压缩的同时，更新节点到根的最大/次大边权，这是算法的核心难点。

时间复杂度

总时间复杂度 $O(m\alpha(n))$ ，其中 $\alpha(n)$ 为阿克曼函数的反函数，近似为常数，效率优于倍增法，适合处理 $n\le 2e5$ 的大规模数据，缺点是必须离线处理所有查询。

竞赛可用完整模板（严格次小生成树，Tarjan离线版）

第四部分 经典例题精讲

例题1 洛谷 P4180 [BJWC2010] 严格次小生成树

题目链接

https://www.luogu.com.cn/problem/P4180

题目难度

提高+/省选-

核心考点

严格次小生成树、倍增LCA、树上路径最大/次大边权查询

题目大意

给定一个 $n$ 个点 $m$ 条边的无向连通带权图，求其严格次小生成树的总权值。

解题思路

  1. 用Kruskal算法求出原图的MST，构建MST的邻接表，标记树边与非树边；

  2. 用倍增法预处理MST上每个节点的倍增数组，以及路径的最大/次大边权；

  3. 枚举所有非树边，查询边的两个端点在MST上的路径的最大/次大边权，计算替换后的权值；

  4. 所有替换后的权值的最小值即为答案。

关键坑点

  • 必须处理边权相等的情况，此时需要用次大边权替换，才能保证严格次小；

  • 数据范围较大，总权值必须用long long存储，避免溢出；

  • $n\le 1e5$ 时，倍增数组的LOG需设为20，避免数组越界。

例题2 Codeforces 160D Edges in MST

题目链接

https://codeforces.com/contest/160/problem/D

题目难度

Div.2 D，Rating 1900

核心考点

MST的核心性质、次小生成树原理、Tarjan求桥

题目大意

给定一个无向连通带权图，对于每条边，判断其属于以下哪一类：

  1. any：所有MST都包含这条边（必选边）；

  2. at least one：存在至少一个MST包含这条边，但不是所有MST都包含（可选边）；

  3. none：任何MST都不包含这条边（不可选边）。

解题思路

  1. 按边权从小到大排序，将边权相同的边分为一组，按Kruskal的顺序处理；

  2. 对于同一组边，先判断哪些边连接的两个顶点已经连通，这些边违反回路性质，属于none；

  3. 对于剩下的边，构建临时图：将并查集的连通块缩点，边为当前组的边；

  4. 对临时图求桥，桥边是连接两个连通块的唯一最小边，属于any；非桥边是同权可选边，属于at least one；

  5. 处理完一组边后，将所有边加入并查集，合并连通块。

例题3 洛谷 P2619 [国家集训队]Tree I

题目链接

https://www.luogu.com.cn/problem/P2619

题目难度

省选/NOI-

核心考点

二分答案、Kruskal算法、带边数限制的MST

题目大意

给定一个无向连通带权图，边分为黑白两种，求恰好包含 $k$ 条白边的MST的最小总权值。

解题思路

  1. 二分白边的权值偏移量mid，给所有白边的权值加上mid；

  2. 用Kruskal求MST，权值相同时优先选白边，记录MST中白边的数量cnt；

  3. 若cnt >= k，说明偏移量太小，白边优先级过高，需要增大mid；否则需要减小mid；

  4. 二分找到合适的偏移量，计算最终的MST总权值，减去k * mid即为答案。

核心原理

通过二分偏移量，调整白边在排序中的优先级，从而控制MST中白边的数量，利用MST的边权单调性保证二分的正确性。

第六部分 学习路径与避坑指南

学习路径建议

  1. 基础阶段：先掌握Kruskal和Prim的基础实现，刷完入门级所有题目，确保能熟练处理MST的基础问题，理解MST的核心性质；

  2. 进阶阶段：学习倍增法实现严格次小生成树，吃透P4180模板题，理解替换原理和路径边权合并逻辑，刷完进阶级题目，掌握MST性质的常见考法；

  3. 拔高阶段：学习Tarjan离线法，对比倍增法的优缺点，掌握带权并查集的路径压缩维护技巧，挑战提高级题目，结合二分、LCT、Trie等知识点，提升综合解题能力。

竞赛避坑指南

  1. 数据溢出：MST总权值通常会超过int的范围，必须用long long存储，尤其是边权较大、点数较多的题目；

  2. 严格次小的边界：处理边权相等的情况时，必须维护次大边权，不能只用最大边权，否则会得到非严格次小生成树，导致答案错误；

  3. 倍增数组大小： $n\le 1e5$ 时，LOG必须设为20， $n\le 2e5$ 时设为21，避免数组越界；

  4. Tarjan带权并查集：路径压缩时必须正确维护最大/次大边权，否则会导致查询结果错误；

  5. 图不连通的情况：必须判断MST的边数是否为 $n-1$ ，避免处理不连通图时出现错误。
