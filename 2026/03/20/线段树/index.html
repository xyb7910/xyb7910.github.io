一、线段树的结构和思想

1.1 核心思想

线段树是一种二叉搜索树，将一个长度为$n$的数组（区间）不断二分，分解为若干个互不相交的子区间，每个节点对应一个子区间，叶子节点对应数组的单个元素。

通过对节点的信息维护和合并，实现对区间的高效修改与查询，将原本暴力操作的$O(n)$时间复杂度优化为$O(\log n)$。

1.2 结构特点

  1. 根节点对应整个数组区间$[1,n]$；
  2. 每个非叶子节点$[l,r]$会被二分为左子节点$[l,mid]$和右子节点$[mid+1,r]$（$mid=(l+r)/2$）；
  3. 叶子节点对应$[i,i]$，存储数组第$i$个元素的原始信息；
  4. 每个节点存储对应区间的聚合信息（如最小值、最大值、和、最大子段和等），聚合信息可通过左右子节点的信息合并得到；
  5. 线段树的空间开法：通常开数组长度的4倍（$4×n$），避免越界。

1.3 核心操作框架

线段树的所有操作均基于递归实现，核心包含三大基础函数：

  1. build：构建线段树，从根节点开始递归分解区间，初始化每个节点的聚合信息；
  2. modify：修改操作（单点/区间），递归找到目标区间/位置，更新节点信息并回溯合并；
  3. query：查询操作，递归分解查询区间，合并子区间的聚合信息得到查询结果。

二、基础操作：单点修改与区间查询（例题1：线段树1）

2.1 例题描述

给定$n$个数，支持两种操作：

  1. 1 x d：将第$x$个数修改为$d$；
  2. 2 l r：查询区间$[l,r]$的最小值，以及最小值出现的次数。
    输入范围：$1≤n,q≤2×10^5$，$1≤a_i,d≤10^9$，输出最小值和其出现次数。

2.2 核心思路

  1. 聚合信息设计：每个节点存储两个信息——区间最小值minv、最小值出现次数mincnt；
  2. 信息合并规则：
    • 若左子节点最小值 < 右子节点最小值，父节点最小值=左子节点最小值，次数=左子节点次数；
    • 若左子节点最小值 > 右子节点最小值，父节点最小值=右子节点最小值，次数=右子节点次数；
    • 若相等，父节点最小值=该值，次数=左子节点次数+右子节点次数；
  3. 单点修改：递归找到目标叶子节点，更新其minv和mincnt，然后自底向上回溯更新所有父节点的聚合信息；
  4. 区间查询：递归分解查询区间，将查询区间拆分为若干个线段树节点的区间，合并这些节点的聚合信息得到结果。

2.3 关键代码实现（C++）

2.4 易错点

  1. 线段树空间需开4倍$n$，否则会出现数组越界；
  2. 递归边界：叶子节点$l==r$的处理；
  3. 单点修改后必须回溯更新所有父节点，否则信息失效；
  4. 区间查询的拆分逻辑：需严格判断查询区间与当前节点区间的包含关系。

三、进阶操作：复杂的信息合并（例题2：线段树2）

3.1 例题描述

给定$n$个数，支持两种操作：

  1. 1 x d：将第$x$个数修改为$d$；
  2. 2 l r：查询区间$[l,r]$的最大子段和（找到$l≤i≤j≤r$，使得$a_i+a_{i+1}+…+a_j$最大）。
    输入范围：$1≤n,q≤2×10^5$，$-10^9≤a_i,d≤10^9$，输出最大子段和。

3.2 核心思路

最大子段和无法通过简单的区间最值/和合并，需设计更复杂的聚合信息，每个节点存储四个关键信息，才能实现区间合并：

  1. mss：当前区间的最大子段和；
  2. mpre：当前区间以左端点为起点的最大子段和；
  3. msuf：当前区间以右端点为终点的最大子段和；
  4. s：当前区间的总和。

信息合并规则

对于父节点$[l,r]$，左子节点$L[l,mid]$，右子节点$R[mid+1,r]$：

  1. 总和：$father.s = L.s + R.s$；
  2. 左起点最大子段和：$father.mpre = max(L.mpre, L.s + R.mpre)$（要么是左子树的左起点，要么是左子树全部+右子树的左起点）；
  3. 右终点最大子段和：$father.msuf = max(R.msuf, R.s + L.msuf)$（要么是右子树的右终点，要么是右子树全部+左子树的右终点）；
  4. 最大子段和：$father.mss = max(L.mss, R.mss, L.msuf + R.mpre)$（三者取最大：左子树最大、右子树最大、跨左右子树的最大）。

3.3 关键代码实现（C++）

3.4 核心要点

  1. 复杂信息合并的关键是设计合适的聚合信息，确保父节点的信息能完全由左右子节点推导；
  2. 针对负数情况，聚合信息的初始化需合理（如单个负数的最大子段和为自身）；
  3. 利用C++构造函数简化叶子节点的初始化，提高代码可读性。

四、高级操作：区间打标记（懒标记）与标记下传

4.1 懒标记核心思想

线段树的单点修改效率为$O(\log n)$，但直接进行区间修改（如区间加、区间乘）时，若逐个修改叶子节点，时间复杂度会退化为$O(n\log n)$，无法满足大数据范围要求。

懒标记（Lazy Tag） 是线段树的核心优化技巧：

  1. 当修改操作覆盖整个线段树节点的区间时，不立即递归修改其子节点，而是在该节点上记录一个标记，表示该节点的所有子节点需要执行此修改操作；
  2. 当后续的修改/查询操作需要访问该节点的子节点时，再将标记下传（pushdown） 到子节点，更新子节点的信息和标记，然后清空当前节点的标记；
  3. 懒标记的本质是延迟更新，将多次区间修改的操作合并，仅在必要时执行，保证时间复杂度仍为$O(\log n)$。

4.2 基础懒标记：区间加法+区间查询最大值（例题3：线段树打标记1）

4.2.1 例题描述

给定$n$个数，支持两种操作：

  1. 1 l r d：将区间$[l,r]$的所有数加上$d$；
  2. 2 l r：查询区间$[l,r]$的最大值。
    输入范围：$1≤n,q≤2×10^5$，$1≤a_i,d≤10^9$，输出区间最大值。

4.2.2 核心思路

  1. 聚合信息：每个节点存储区间最大值maxv；
  2. 懒标记设计：每个节点存储一个加法标记add，表示该节点的所有子节点需要加上add，初始值为0；
  3. 标记操作：
    • settag：给节点打标记，更新节点的最大值（区间加$d$，最大值也加$d$），并将标记累加到节点的add上；
    • pushdown：将当前节点的标记下传到子节点，执行settag更新子节点，然后清空当前节点的标记；
  4. 区间修改：递归找到目标区间，若当前节点区间完全被覆盖则打标记，否则先下传标记再递归修改子节点，最后回溯更新当前节点信息；
  5. 区间查询：与基础查询一致，但查询前需先下传标记，确保访问的子节点信息是最新的。

4.3 复杂懒标记：加法、乘法、赋值混合操作（例题4：线段树打标记2）

4.3.1 例题描述

给定$n$个数，支持四种操作：

  1. 1 l r d：区间$[l,r]$加$d$；
  2. 2 l r d：区间$[l,r]$乘$d$；
  3. 3 l r d：区间$[l,r]$赋值为$d$；
  4. 4 l r：查询区间$[l,r]$的和，结果对$10^9+7$取模。
    输入范围：$1≤n,q≤2×10^5$，$1≤a_i,d≤10^9$，输出区间和取模结果。

4.3.2 核心思路

这是懒标记的经典复杂场景，多种标记混合时，需解决标记的优先级和标记的合并规则问题：

  1. 聚合信息：每个节点存储区间和sum，以及区间长度len（赋值操作需要）；
  2. 懒标记设计：每个节点存储三个标记，但为了简化，将所有操作统一为线性变换：$a[i] = a[i] × mul + add$，其中：
    • mul：乘法标记，初始值为1；
    • add：加法标记，初始值为0；
    • 赋值操作$d$可转化为：$mul=0$，$add=d$（$a[i]×0 + d = d$）；
  3. 标记优先级：赋值 > 乘法 > 加法，打标记时需按此顺序合并；
  4. 标记合并规则：若当前节点已有标记$(mul1, add1)$，再施加新标记$(mul2, add2)$，合并后为：
    $$mul = mul1 × mul2, \quad add = add1 × mul2 + add2$$
  5. 标记下传：将当前节点的线性变换标记下传到子节点，更新子节点的sum、mul、add，然后清空当前节点标记。

4.3.3 核心要点

  1. 多种操作混合时，统一为线性变换是最优解，避免单独维护多个标记的复杂逻辑；
  2. 区间和的更新规则：若区间执行$x = x×mul + add$，则区间和$sum = sum×mul + add×len$（$len$为区间长度）；
  3. 打标记前必须先下传已有标记，确保标记的合并顺序正确；
  4. 所有操作需取模，避免数值溢出。

五、特殊应用：线段树上二分（例题5：线段树上二分）

5.1 例题描述

给定$n$个数，支持两种操作：

  1. 1 x d：将第$x$个数修改为$d$；
  2. 2 l r d：查询区间$[l,r]$中大于等于$d$的第一个数的下标，若不存在则输出-1。
    输入范围：$1≤n,q≤2×10^5$，$1≤a_i,d≤10^9$，输出最小的满足条件的下标。

5.2 核心思路

线段树上二分是线段树的经典拓展应用，利用线段树的区间聚合信息和二分特性，在$O(\log n)$时间内找到满足条件的位置，核心要求：区间的聚合信息能判断其子区间是否存在满足条件的元素。

本题的实现步骤：

  1. 聚合信息：每个节点存储区间的最大值，用于判断该区间是否存在≥$d$的元素；
  2. 二分逻辑（查询$[l,r]$中第一个≥$d$的元素）：
    • 递归到当前节点，若节点区间与$[l,r]$无交集，返回-1；
    • 若节点是叶子节点，判断是否≥$d$，是则返回下标，否则返回-1；
    • 先检查左子节点：若左子节点与$[l,r]$有交集且左子节点最大值≥$d$，则递归查询左子节点（保证找第一个）；
    • 若左子节点无满足条件的元素，再递归查询右子节点；
    • 若均无，返回-1；
  3. 单点修改：与基础线段树一致，更新叶子节点并回溯更新最大值。

5.3 关键代码实现（C++）

5.4 拓展场景

线段树上二分可应用于各类找第一个/最后一个满足条件的元素的问题，例如：

  1. 找区间中第一个小于$d$的元素；
  2. 找区间中最后一个大于$d$的元素；
  3. 结合前缀和，找区间中前缀和≥$k$的第一个位置。
    核心不变：设计合适的聚合信息，让每个节点能快速判断子区间是否存在满足条件的元素。

六、知识点总结

6.1 线段树核心要素

  1. 聚合信息设计：根据问题需求设计，是线段树的核心，需满足可合并性（父节点信息可由左右子节点推导）；
  2. 基础操作：build（构建）、modify（修改）、query（查询），均基于递归实现；
  3. 懒标记优化：用于区间修改，核心是延迟更新，需实现settag（打标记）和pushdown（标记下传），多种标记混合时需统一为线性变换；
  4. 线段树上二分：利用区间聚合信息的判断特性，在$O(\log n)$时间内找满足条件的位置，保证先查左子树找第一个，后查右子树找最后一个。

6.2 时间复杂度

线段树的所有操作（构建、单点修改、区间修改、区间查询、树上二分）的时间复杂度均为$O(\log n)$，对于$q$次操作，总时间复杂度为$O(n + q\log n)$，能满足$n,q≤2×10^5$的大数据范围要求。

6.3 空间复杂度

线段树的空间复杂度为$O(4n)$，是典型的空间换时间的算法。

6.4 适用场景

线段树适用于一维数组的区间修改和区间查询问题，尤其适合：

  1. 单点修改 + 区间查询；
  2. 区间修改 + 区间查询；
  3. 找区间中满足条件的位置；
  4. 复杂信息的区间合并（如最大子段和、区间众数等）。

6.5 易错点汇总

  1. 线段树空间未开4倍$n$，导致数组越界；
  2. 递归边界处理错误（未判断$l==r$）；
  3. 修改操作后未回溯更新父节点信息；
  4. 懒标记下传时机错误（访问子节点前未下传）；
  5. 多种懒标记混合时，合并规则和优先级错误；
  6. 线段树上二分时，未先查左子树导致找的不是第一个满足条件的元素。

6.6 与树状数组的对比

线段树和树状数组都是处理区间问题的高效数据结构，二者对比：

特性 线段树 树状数组
适用场景 几乎所有区间问题 仅支持前缀和相关操作
实现难度 较高（复杂信息/懒标记） 较低
时间复杂度 $O(\log n)$ $O(\log n)$
空间复杂度 $O(4n)$ $O(n)$
拓展性 强（可处理二维/树上） 弱（仅一维前缀和）

结论：树状数组能解决的问题优先用树状数组，树状数组无法解决的（如区间最值、最大子段和、复杂区间修改）用线段树。
