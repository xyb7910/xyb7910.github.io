拓扑排序深入应用

拓扑图的深入应用

一、有向图判环与环定位

这是拓扑排序最基础的进阶用法，也是所有有向图问题的前置校验——绝大多数有向图算法的前提是图无环，而拓扑排序是最直观、最不易写错的判环方案。

1.1 核心原理

拓扑排序的本质是“不断消除无前置依赖的节点（入度为0）”，而有向环的核心特征是：环上的所有节点互相依赖，永远不存在入度为0的起点。

  • 环上的每个节点，入度至少有1来自环内的另一个节点，无论怎么消除环外的依赖，环上节点的入度永远无法降到0，永远无法进入拓扑序列。

  • 因此：拓扑排序结束后，若拓扑序列的长度小于总顶点数，则图中一定存在有向环。

1.2 两种实现方案详解

方案1：Kahn算法判环

完整实现步骤

  1. 建图与初始化：邻接表存图，维护in_degree[]数组记录每个节点的入度；

  2. 初始化队列：遍历所有节点，将入度为0的节点加入队列；

  3. 拓扑遍历：取出队首节点，加入拓扑序列，遍历其所有出边，将对应邻接节点的入度减1，若入度降至0则加入队列；

  4. 判环逻辑：遍历结束后，若拓扑序列的长度tot < n（总顶点数），则图存在有向环；否则为无环DAG。

环定位

很多初学者只会判环，不会精准找到环上的节点，这里拆解两种定位需求的实现：

  1. 定位所有受环影响的节点：Kahn算法结束后，所有in_degree[v] > 0的节点，要么在环上，要么在环的下游（能被环到达的节点）。这些节点永远无法进入拓扑序列，因为环的存在导致它们的入度无法清零。

  2. 精准定位环上的节点：

    • 第一步：执行Kahn算法，标记所有进入拓扑序列的节点（vis[u] = 1），未被标记的节点是“环+环的下游节点”；

    • 第二步：对未被标记的节点，求强连通分量（SCC）。大小≥2的SCC就是有向环，大小为1的SCC若存在自环（u→u）也是环。

    • 补充：若只需快速输出环的路径，用DFS方案更直接。

带环定位的代码实现

方案2：DFS逆序法判环（优势：直接输出环的路径）

DFS判环的核心是通过节点访问状态标记，识别递归过程中的环，优势是可以直接输出环的完整路径，适合需要定位环具体内容的场景。

核心状态定义

用vis[]数组标记节点的3种访问状态，是DFS判环的核心，绝对不能写错：

  • vis[u] = 0：节点未被访问；

  • vis[u] = 1：节点正在访问（处于当前递归栈中）；

  • vis[u] = 2：节点已访问完毕（所有子节点都处理完成，已退出递归栈）。

判环与环路径输出逻辑

  • 遍历节点u的出边u→v时，若vis[v] == 1，说明我们从v出发，递归后又回到了v，存在有向环；

  • 环的路径：当前递归栈中，从v到当前节点u的所有节点，就是环的完整路径。

1.3 两种方案的竞赛场景选择

场景 首选方案 原因
仅需判环，无需找环路径 Kahn算法 代码简洁，不易写错，可同步得到拓扑序
需要输出环的具体路径 DFS方案 可直接通过递归栈定位环，无需额外求SCC
大数据量（n≥1e5） Kahn算法 递归DFS有栈溢出风险，需改非递归，Kahn更稳定
多环场景 Kahn+SCC 可批量定位所有环，DFS需多次处理

1.4 常见坑点避坑

  1. 自环处理：节点u存在自环u→u时，in_degree[u] ≥ 1，Kahn会正确判环，DFS也会识别，注意题目是否允许自环；

  2. 环的下游节点误区：Kahn中in_degree>0的节点不都是环上的点，仅能说明它们受环影响，不能直接当作环节点；

  3. DFS状态标记错误：必须用3种状态，仅用“访问/未访问”两种状态会误判环；

  4. 非连通图处理：必须遍历所有节点，不能只从一个起点DFS/初始化队列，否则会漏判不连通的环。

二、DAG上的动态规划

这是拓扑排序最核心、最高频的竞赛应用，没有之一。DAG上的所有动态规划问题，都必须依赖拓扑排序保证转移的合法性，是省选及以下难度的必考考点。

2.1 核心原理：拓扑排序解决DP的无后效性问题

动态规划的核心要求是无后效性：计算当前状态时，所有能转移到它的前驱状态必须已经计算完毕，不能出现“当前状态依赖未来未计算的状态”。

而DAG的拓扑序，完美满足这个要求：

对于DAG中任意一条有向边u→v（表示v的状态可由u的状态转移而来），在拓扑序列中，u一定出现在v的前面。

也就是说，当我们按拓扑序的顺序计算v的状态时，u的状态已经100%计算完成，绝对不会出现后效性。反之，若不按拓扑序随机转移，大概率会得到错误结果。

2.2 DAG上DP的通用万能步骤

竞赛中所有DAG上的DP问题，都可以按这个固定流程解决，不会出现逻辑错误：

  1. 问题建模：把题目中的“状态”转化为DAG的顶点，“状态转移关系”转化为DAG的有向边；

  2. 拓扑排序：对DAG执行Kahn算法，得到合法的拓扑序列；

  3. DP初始化：根据题意，给初始状态（通常是入度为0的顶点，拓扑序最前端的点）赋正确的初始值；

  4. 按拓扑序转移：遍历拓扑序列中的每个顶点u，遍历u的所有出边u→v，用u的DP值更新v的DP值；

  5. 统计答案：根据题意，从所有顶点的DP值中统计最终结果（最大值、总和、终点值等）。

2.3 竞赛高频DP类型详解

下面拆解竞赛中最常考的3类DAG上的DP，每个都包含原理、例题、核心代码和避坑指南。

类型1：DAG上的最长路/最短路

这是最基础、最高频的考点，也是所有DAG DP的基础。

为什么不用Dijkstra/SPFA，而用拓扑排序？

  1. 时间复杂度更优：拓扑排序是O(n+m)线性时间，比Dijkstra的O(m log n)更快，且不会被卡；

  2. 无负权限制：普通Dijkstra无法处理正权最长路，SPFA处理最长路容易被卡，而拓扑排序无论边权正负，只要是DAG就能完美处理；

  3. 逻辑更统一：最长路和最短路的代码几乎完全一致，仅需修改max/min和初始值，不易写错。

最长路实现详解

  • 问题模型：给定DAG，边权为w[u][v]，求图中的最长路径（DAG直径），或从起点s到终点t的最长路径长度。

  • 状态定义：dp[u]表示从起点到u的最长路径长度；若求全图最长路，dp[u]表示以u为终点的最长路径长度。

  • 转移方程：对于边u→v，dp[v] = max(dp[v], dp[u] + w[u][v])。

  • 初始化：单源最长路则dp[s] = 0，其余为-INF；全图最长路则所有入度为0的点dp[u] = 0，其余为-INF。

  • 转移顺序：严格按拓扑序正序遍历。

最短路实现详解

  • 状态定义：dp[u]表示从起点到u的最短路径长度。

  • 转移方程：对于边u→v，dp[v] = min(dp[v], dp[u] + w[u][v])。

  • 初始化：单源最短路则dp[s] = 0，其余为INF。

  • 转移顺序：同样按拓扑序正序遍历。

经典例题：洛谷 P1113 杂物

  • 题意：n个杂物，每个杂物有完成时长，且必须完成所有前置杂物才能开始，求完成所有杂物的最短时间。

  • 建模：杂物为顶点，前置关系u是v的前置转化为边u→v，边权为v的完成时长；dp[u]表示完成u的最早结束时间。

  • 转移方程：dp[v] = max(dp[v], dp[u] + time[v])（v的最早开始时间是所有前置的最晚结束时间，加自身时长就是结束时间）。

类型2：DAG上的计数DP

高频考点，常考路径计数、方案计数、可达性计数等，比如洛谷P4017 最大食物链计数。

核心实现逻辑

  • 问题模型：求DAG中从起点到终点的路径总数，或满足条件的方案数。

  • 状态定义：dp[u]表示从起点到u的路径数量。

  • 转移方程：对于边u→v，dp[v] = (dp[v] + dp[u]) % MOD（所有到u的路径，都可以延伸到v）。

  • 初始化：起点s的dp[s] = 1；若所有入度为0的点都是起点，全部设为1。

  • 转移顺序：拓扑序正序遍历。

经典例题：洛谷 P4017 最大食物链计数

  • 题意：给定食物网（DAG），边u→v表示u被v吃，求完整的食物链数量（从最底层生产者到最高级捕食者的路径数）。

  • 建模：生产者是入度为0的点，捕食者是出度为0的点；dp[u]表示以u为终点的食物链数量。

  • 转移方程：dp[v] = (dp[v] + dp[u]) % MOD。

  • 答案：所有出度为0的点的dp值之和。

类型3：DAG上的概率/期望DP

竞赛中进阶考点，拓扑排序是解决这类问题的最优方案，避免了DFS递归的栈溢出和重复计算问题。

核心逻辑

概率/期望DP的核心是“当前状态的期望，由所有前驱状态的期望加权求和得到”，拓扑序保证了计算当前状态时，所有前驱的期望已经计算完成。

  • 状态定义：dp[u]表示到达u的概率/期望；

  • 转移方程：对于边u→v，dp[v] += dp[u] * p(u→v)（p是转移概率）；

  • 初始化：起点的dp[s] = 1（概率）或对应初始期望；

  • 转移顺序：拓扑序正序遍历。

2.4 常见坑点避坑

  1. 转移顺序搞反：必须按拓扑序正序转移，不能逆序，否则前驱状态未计算完成，结果全错；

  2. 初始化错误：入度为0的点的初始值必须正确，比如最长路初始化为0，而非-INF，否则会导致所有转移失效；

  3. 有环图直接DP：必须先确保图是DAG，有环的话要先缩点，否则拓扑排序失败，DP有后效性；

  4. 用DFS代替拓扑序：DAG深度大时，递归DFS会栈溢出，且重复计算会超时，竞赛中优先用拓扑序的迭代写法；

  5. 边权处理错误：最长路要注意负权边的初始值，最短路要注意正权边的初始值，避免溢出。

三、强连通分量缩点后的DAG处理

这是竞赛有向图难题的万能套路，90%的有环有向图问题，都可以通过“缩点+拓扑排序”转化为简单的DAG问题解决，是省选难度的核心考点。

3.1 核心原理

有向图中的强连通分量（SCC） 有一个关键特性：同一个SCC内的任意两个节点，都可以互相到达。

  • 对于很多问题，同一个SCC内的节点可以合并处理（比如点权求和、路径计数时，SCC内的点可以互相到达，贡献一致）；

  • 把每个SCC缩成一个“超级点”，缩点后的图一定是DAG（如果缩点后还有环，环上的超级点可以合并成一个更大的SCC，与SCC的极大性矛盾）。

这个操作，就把复杂的有环有向图，转化为了我们熟悉的DAG，之后就可以用拓扑排序+DP解决所有问题。

3.2 竞赛通用标准流程

这个流程是固定的，必须熟练掌握，所有缩点类题目都按这个步骤来：

  1. 求SCC：对原图用Tarjan算法或Kosaraju算法，求出所有强连通分量，给每个节点u标记所属的SCC编号bel[u]；

  2. 缩点建图：

    • 统计每个SCC的信息：比如SCC的大小、SCC内所有点的权值和、SCC内的节点数量等，根据题目需求调整；

    • 构建缩点后的DAG：遍历原图的所有边u→v，若bel[u] != bel[v]，说明这是两个SCC之间的边，在缩点后的图中加边bel[u] → bel[v]，必须去重，避免重复加边导致入度统计错误；

  3. 拓扑排序：对缩点后的DAG执行Kahn算法，得到拓扑序列；

  4. 问题求解：在缩点后的DAG上，按拓扑序做DP、统计、路径等操作，得到原问题的答案。

3.3 高频竞赛应用场景详解

场景1：有向图的最大权路径（模板题：洛谷 P3387 缩点）

这是缩点+拓扑排序最经典的应用，也是竞赛必背模板题。

  • 题意：给定有向图，每个点有权值，路径上的点权只算一次，求点权和最大的路径。

  • 问题难点：图中存在环，环上的点可以全部走到，点权可以全部拿到，直接DP有后效性。

  • 解法步骤：

    1. Tarjan求SCC，每个SCC的权值为内部所有点的权值之和；

    2. 缩点建DAG，去重边，统计每个超级点的入度；

    3. 对DAG拓扑排序，按拓扑序求最长路，dp[u]表示到达超级点u的最大权值和；

    4. 所有dp[u]的最大值就是答案。

场景2：有向图可达性统计

  • 问题：给定n=1e4、m=1e5的有向图，求每个点能到达的节点数量，暴力DFS每个点会超时。

  • 解法：

    1. 缩点后，同一个SCC内的点可达性完全一致，只需统计每个超级点的可达性；

    2. 对缩点后的DAG按逆拓扑序遍历，用bitset统计每个超级点能到达的超级点集合；

    3. 每个点的可达数量 = 所属SCC的大小 + 可达的其他SCC的大小之和。

  • 优势：缩点后点数大幅减少，bitset优化后时间复杂度降至O(n+m + n²/64)，可处理1e4规模的数据。

场景3：有向图的必经点/必经边

缩点后，DAG上的必经点/必经边可以通过拓扑序+逆拓扑序的路径计数快速求解，是竞赛中进阶考点。

3.4 常见坑点避坑

  1. 缩点后加边不去重：这是初学者最容易犯的致命错误！原图中多条边对应同一个SCC间的边，重复加边会导致入度统计错误，拓扑排序完全失效。必须用set或哈希去重；

  2. SCC信息统计错误：bel数组标记错误、SCC权值和算错，会导致后续DP全错，必须先验证SCC求解的正确性；

  3. 拓扑排序的对象错误：缩点后必须对新的DAG跑拓扑排序，而不是原图，很多初学者会在这里搞混；

  4. Tarjan算法实现错误：low值更新错误、栈操作错误，会导致SCC求解错误，是整个流程的基础，必须先把Tarjan模板写对。

四、差分约束系统的拓扑解法

差分约束系统是竞赛中高频的不等式组求解问题，绝大多数初学者只知道用SPFA求解，却不知道当约束对应的图是DAG时，拓扑排序解法效率更高、更稳定，不会被卡。

4.1 差分约束系统基础

差分约束系统，就是给定n个变量x₁~xₙ，和m个形如xᵢ - xⱼ ≤ c（或≥、=）的不等式，求变量的一组可行解，或某个变量的最大值/最小值。

核心是不等式到图的边的转化，所有约束都可以转化为有向边：

不等式 变形 对应图的边 对应模型
xᵢ - xⱼ ≤ c xᵢ ≤ xⱼ + c j → i，边权c 最短路模型
xᵢ - xⱼ ≥ c xⱼ ≤ xᵢ - c i → j，边权-c 最短路模型
xᵢ - xⱼ ≥ c xᵢ ≥ xⱼ + c j → i，边权c 最长路模型
xᵢ = xⱼ xᵢ ≤ xⱼ+0 且 xⱼ ≤ xᵢ+0 j→i(0)、i→j(0) 通用
转化完成后，问题就变成了图的单源最短/最长路问题：

  • 求变量的最大值：用最短路模型；

  • 求变量的最小值：用最长路模型；

  • 无解情况：图中存在负环（最短路）或正环（最长路），说明不等式组矛盾。

4.2 拓扑排序解法的核心逻辑

如果转化后的图是DAG，那么完全不需要用SPFA，直接用拓扑排序求解最短/最长路，优势极大：

  • 时间复杂度O(n+m)，线性时间，比SPFA快得多，且不会被卡；

  • 无需判环，DAG天然无环，不会出现不等式矛盾的情况；

  • 代码更简洁，没有SPFA的队列优化和负环判断，不易写错。

4.3 完整实现步骤

  1. 不等式转化：把所有约束转化为对应的有向边，构建图；

  2. 建立超级源点：差分约束系统需要一个统一的起点，通常建立超级源点0，对每个变量xᵢ，加边0→i，边权根据题目需求设置（比如要求xᵢ≥1，边权设为1；无特殊要求设为0）；

  3. 拓扑排序：对包含超级源点的图跑Kahn算法，得到拓扑序列；

  4. 按拓扑序求解：

    • 最长路模型（求最小值）：初始化dis[0] = 0，其余为-INF，按拓扑序正序遍历，对边u→v，dis[v] = max(dis[v], dis[u] + w[u][v])；

    • 最短路模型（求最大值）：初始化dis[0] = 0，其余为INF，按拓扑序正序遍历，对边u→v，dis[v] = min(dis[v], dis[u] + w[u][v])；

  5. 输出解：dis[1]~dis[n]就是变量x₁~xₙ的一组可行解。

4.4 经典例题：洛谷 P1983 [NOIP2013 普及组] 车站分级

  • 题意：火车经过的车站中，停靠的车站等级必须严格大于不停靠的车站等级，求最少需要多少个等级。

  • 建模：

    • 变量a[i]表示第i个车站的等级，要求最小值，用最长路模型；

    • 对于每个车次，停靠的车站集合为S，不停靠的为T，对任意u∈T、v∈S，有a[v] ≥ a[u] + 1，转化为边u→v，边权1；

    • 建立超级源点0，加边0→i，边权1（每个车站等级至少为1）。

  • 解法：转化后的图是DAG，直接拓扑排序求最长路，所有dis[i]的最大值就是最少需要的等级数。

4.5 与SPFA的场景选择

场景 首选方案
转化后的图是DAG（约束无环，如层级、先后顺序） 拓扑排序解法
图中存在环，需要判环找矛盾 SPFA
大数据量，无环 拓扑排序解法，避免SPFA被卡

4.6 常见坑点避坑

  1. 不等式转化错误：把≤和≥搞反，导致边的方向和权值错误，结果全错，这是最常见的致命错误；

  2. 忘记建立超级源点：导致图不连通，很多节点的dis值无法更新，得到错误的解；

  3. 最长路和最短路模型搞混：求变量的最小值必须用最长路模型，求最大值必须用最短路模型，初学者极易搞反；

  4. 边权符号错误：不等式变形时，边权的正负号写错，导致约束完全相反。

五、2-SAT问题中的拓扑排序应用

2-SAT是省选及以上难度的高频考点，拓扑排序（逆拓扑序）是2-SAT问题有解时赋值的核心依据，没有拓扑排序，就无法得到合法的变量赋值。

5.1 2-SAT问题基础

2-SAT问题：给定n个布尔变量（每个变量只能取true/false），和m个二元约束（每个约束仅涉及两个变量），求变量的一组合法赋值，或判断无解。

核心拆点与建图规则

把每个布尔变量a拆成两个节点：

  • 2a：表示a取false（¬a）；

  • 2a+1：表示a取true（a）。

每个约束转化为两条有向边（逆否命题），比如最常见的约束a∨b（a为真或b为真），等价于：

  • 若a为假，则b必须为真：¬a → b → 边2a → 2b+1；

  • 若b为假，则a必须为真：¬b → a → 边2b → 2a+1。

无解判定

用Tarjan算法对建图后的有向图求SCC，若对于某个变量a，2a和2a+1在同一个SCC中，说明a既要为真又要为假，矛盾，问题无解。

5.2 拓扑排序在2-SAT中的核心作用

若问题有解，如何给变量赋值？这里必须用到缩点后DAG的逆拓扑序，这是2-SAT赋值的唯一正确方法。

为什么必须用逆拓扑序？

缩点后的图是DAG，边u→v的含义是“若u为真，则v必须为真”。

  • 若按正拓扑序赋值：先给上游节点赋值，可能出现“上游赋值为真，导致下游必须为真，但下游已经被赋值为假”的冲突；

  • 若按逆拓扑序赋值：先给下游节点赋值，再给上游节点赋值，保证赋值时，当前节点的所有后继（出边指向的节点）已经赋值完毕，绝对不会出现约束冲突。

赋值规则（必须牢记）

对于每个变量a，设bel[2a]是¬a所在的SCC编号，bel[2a+1]是a所在的SCC编号：

a = (bel[2a+1] > bel[2a])

这个规则的本质是：选择SCC编号更大的那个取值。因为Tarjan算法求SCC时，SCC的编号顺序就是缩点后DAG的逆拓扑序——编号越大的SCC，在逆拓扑序中越靠前，赋值优先级越高。

5.3 2-SAT完整实现流程（含拓扑排序）

  1. 拆点：每个变量a拆为2a（¬a）和2a+1（a）；

  2. 建图：把每个约束转化为对应的两条有向边；

  3. 求SCC：用Tarjan算法求所有SCC，标记每个节点的bel编号；

  4. 无解判定：遍历每个变量，若bel[2a] == bel[2a+1]，输出无解；

  5. 赋值：按逆拓扑序（Tarjan的SCC编号），对每个变量a，赋值a = (bel[2a+1] > bel[2a])；

  6. 输出结果：输出每个变量的赋值。

5.5 常见坑点避坑

  1. 拆点编号错误：变量的true/false对应的节点编号搞反，导致赋值完全错误，必须严格按“2a为false，2a+1为true”的规则拆点；

  2. 约束转化错误：忘记加逆否命题的边，导致约束不全，结果错误；

  3. 逆拓扑序搞反：用正拓扑序赋值，导致约束冲突，必须用Tarjan的SCC编号（逆拓扑序）赋值；

  4. 空间开小：每个变量拆2个点，边数是约束数的2倍，必须开2倍以上的空间，否则会数组越界。
