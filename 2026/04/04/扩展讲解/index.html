竞赛向

1. 宏定义（#define）

宏定义是 C++ 的预处理指令，用于在编译前进行文本替换，能简化重复代码、提升编写速度。

1.1 常用宏

1.2 注意事项

  • 宏是纯文本替换，务必加括号避免优先级错误（如 #define mul(a,b) (a)*(b)）；

  • 复杂逻辑建议用 inline 函数替代宏（更安全，有类型检查）；

  • 避免用宏定义变量名，防止命名冲突。

2. typedef 类型重命名

typedef 是 C/C++ 中的关键字，用于为已存在的数据类型创建别名，核心作用是简化复杂类型名、提升代码可读性，是算法竞赛中必备的基础技巧。

2.1 核心语法

  • 原类型名：可以是基本数据类型（int、long long）、复合类型（pair、vector、数组指针）、自定义结构体/类；

  • 别名：自定义的简洁名称，竞赛中习惯用短字母、小写组合（如 ll、pii），命名需简洁易记。

2.2 竞赛常用场景及示例

（1）简化基础大类型

竞赛中频繁处理大数运算，long long 与 unsigned long long 出现频率极高，用 typedef 简化后可大幅缩短代码长度，减少输入错误。

原类型 简化别名 代码示例
long long ll（竞赛通用简写） typedef long long ll; ll ans = 1e18;
unsigned long long ull typedef unsigned long long ull; ull hash_val = 2e19;
int i32（可选，适配跨平台场景） typedef int i32; i32 arr[100];

（2）简化复合容器类型

竞赛中频繁使用 pair、vector 等容器组合，其完整类型名冗长（如 vector<pair<int, vector<int>>>），用 typedef 简化后可读性和编写效率翻倍，尤其适合嵌套容器。

原复杂类型 简化别名 代码示例
pair<int, int> pii（竞赛通用简写） typedef pair<int, int> pii; pii edge = {1, 2};
pair<long long, long long> pll typedef pair<ll, ll> pll; pll pos = {1e10, 2e10};
vector<int> vi typedef vector<int> vi; vi v = {1, 2, 3};
vector<pair<int, int>> vpii typedef vector<pii> vpii; vpii edges;
map<int, int> mii typedef map<int, int> mii; mii cnt;

（3）简化迭代器类型

遍历容器时，迭代器类型名冗长（如 vector<int>::iterator），用 typedef 简化后可减少重复输入，尤其在嵌套容器、多层遍历场景中优势明显。

2.3 与宏定义（#define）的核心区别（竞赛必背！）

很多同学易混淆两者，核心逻辑完全不同，竞赛中需严格区分，避免编译错误。

对比维度 typedef（类型别名） #define（文本替换）
作用对象 仅针对数据类型（int、pair、vector 等） 针对任意文本（类型、变量、表达式、代码片段）
本质逻辑 编译器层面的类型重命名，有严格类型检查 预处理层面的纯文本替换，无类型检查
语法特点 别名是合法类型名，可参与类型推导 替换文本为原始字符，优先级易出错（需加括号）
分号要求 末尾必须加分号（语句结束标志） 末尾不加分号（避免替换内容多分号）
示例对比 typedef long long ll; ll a = 10;（合法） #define ll long long; ll a = 10;（非法，多了分号）

2.4 竞赛易错点（重点规避）

  1. 分号陷阱：typedef 是语句，末尾必须加分号，漏写会直接编译报错；而宏定义绝对不能加分号，两者需严格区分。

  2. 命名冲突：避免将别名命名为 C++ 关键字（如 int、char、if）或标准库函数名（如 sort、max、min），否则会导致编译错误。

  3. 嵌套简化顺序：多层嵌套类型需从内到外逐步简化，例如先简化 pair<int, int> 为 pii，再简化 vector<pii> 为 vpii，不可一步到位简化，否则会降低可读性。

  4. 不滥用别名：仅简化频繁使用、类型名冗长的类型（如 long long、pair、容器迭代器），简单类型（如 int、char、double）无需简化，避免降低代码可读性。

2. auto 关键字（C++11+）

auto 用于自动推导变量类型，能简化复杂类型的书写（如 STL 迭代器）。

2.1 用法示例

2.2 注意事项

  • auto 必须初始化（否则无法推导类型）；

  • C++11 中 auto 不能用于函数参数。

3. 范围 for 循环（Range-based for loop，C++11+）

范围 for 用于遍历容器或数组，语法简洁，分为“加 &”和“不加 &”两种形式。

3.1 语法与区别

3.2 适用场景

  • 不加 &：只读遍历小类型（如 int、char）；

  • 加 &：需修改容器元素，或遍历大对象（如 string、自定义结构体）以提升效率。

4. STL 常见容器学习与使用

STL（标准模板库）是算法竞赛的“神器”，提供了封装好的数据结构，能大幅减少代码量。以下补充各类容器高频扩展操作，适配竞赛高频场景。

4.1 序列式容器

（1）vector（动态数组）

  • 特点：随机访问 O(1)，尾部插入删除 O(1)，中间插入删除 O(n)；

  • 常用操作及扩展操作：

操作 功能 时间复杂度 代码示例
push_back(val) 尾部插入元素val O(1) v.push_back(10);
pop_back() 尾部删除元素 O(1) v.pop_back();
[] 随机访问指定位置元素 O(1) cout << v[2];
size() 返回元素个数 O(1) cout << v.size();
clear() 清空所有元素 O(n) v.clear();
insert(pos, val) 在迭代器pos位置插入val O(n) v.insert(v.begin()+2, 5);
erase(pos) 删除迭代器pos位置元素 O(n) v.erase(v.begin()+1);
resize(n) 调整容器大小为n，不足补默认值 O(n) v.resize(8, 0);
reserve(n) 预分配n个元素空间，不改变实际大小 O(1) v.reserve(100);
front()/back() 获取首/尾元素 O(1) cout << v.front() << v.back();

注意事项

  • 避免频繁在中间插入/删除：中间操作时间复杂度为O(n)，数据量较大（如1e5以上）时会超时，优先用vector做尾部操作，中间操作可考虑list；

  • 区分resize和reserve：resize(n)会改变容器实际大小（补默认值），reserve(n)仅预分配空间，不改变大小，避免频繁扩容可提前用reserve；

  • 禁止越界访问：[]操作不做边界检查，空容器或访问下标≥size()时会报RE，不确定下标合法性时，优先用at()（会抛异常，竞赛中可结合empty()判断）；

  • clear()不释放空间：v.clear()仅清空元素，不释放底层内存，若需释放空间，可使用vector<int>().swap(v);。

（2）string（字符串）

  • 特点：类似 vector<char>，但有字符串专用操作，支持拼接、查找、替换等；

  • 常用操作及扩展操作：

操作 功能 时间复杂度 代码示例
+= 拼接字符串 O(len(str)) s += "world";
substr(pos, len) 取子串（起始位置，长度） O(len) s.substr(0, 5);
find(str) 查找子串，返回起始位置（未找到返回-1） O(n*m) s.find("abc");
append(str) 拼接字符串（与+=功能类似） O(len(str)) s.append("123");
replace(pos, len, str) 从pos开始，替换len个字符为str O(n) s.replace(1, 2, "xy");
erase(pos, len) 从pos开始，删除len个字符 O(n) s.erase(0, 1);
compare(str) 与str比较，返回差值（同strcmp） O(n) s.compare("test") == 0;
c_str() 转换为C风格字符串（const char*） O(1) printf("%s", s.c_str());

注意事项

  • find()返回值判断：s.find(str)未找到时返回string::npos，而非-1，用if (s.find(str) == -1)会出错，正确写法是if (s.find(str) == string::npos)；

  • 字符串拼接效率：频繁用+=拼接短字符串效率可接受，拼接大量短字符串时，优先用stringstream，避免多次扩容耗时；

  • c_str()的有效期：s.c_str()返回的指针仅在当前string对象有效，若string被修改（如拼接、删除），指针会失效，避免保存该指针后续使用；

  • substr()参数注意：第二个参数是“长度”而非“结束下标”，如s.substr(1,3)是从下标1开始，取3个字符，而非取到下标3。

4.2 容器适配器

竞赛中最常用的三类适配器为queue（队列）、stack（栈）与priority_queue（优先队列）。其中queue与stack为基础线性适配器，逻辑规则互补。

（1）queue（队列，FIFO）与stack（栈，LIFO）

两者核心特性对比如下，可直观区分使用场景：

核心维度 queue（队列） stack（栈）
访问规则 先进先出（FIFO）：队尾入队，队首出队 后进先出（LIFO）：栈顶入栈，栈顶出栈
底层容器 默认基于deque实现，可指定list 默认基于deque实现，可指定vector/list
访问权限 仅可访问队首、队尾元素，不支持随机访问 仅可访问栈顶元素，不支持随机访问/遍历中间元素
核心操作复杂度 入队、出队、访问首尾均为O(1) 入栈、出栈、访问栈顶均为O(1)
竞赛核心场景 BFS广度优先搜索、排队模拟、滑动窗口 括号匹配、表达式求值、DFS非递归模拟、单调栈
① queue 详细用法
  • 常用操作（核心接口固定，无额外扩展操作，重点掌握）：
操作 功能 时间复杂度 代码示例
push(val) 队尾入队 O(1) q.push(1);
pop() 队首出队（无返回值） O(1) q.pop();
front() 获取队首元素 O(1) cout << q.front();
back() 获取队尾元素 O(1) cout << q.back();
size() 返回元素个数 O(1) cout << q.size();
empty() 判断队列是否为空 O(1) if (q.empty()) {}
注意事项

  • pop()无返回值：严禁写int x = q.pop();，需先通过front()获取队首元素，再执行pop()；

  • 无随机访问能力：无法用[]或迭代器访问中间元素，仅能访问队首和队尾，若需访问中间元素，需换用deque；

  • 清空队列方式：无clear()方法，需用while(!q.empty()) q.pop();手动清空；

② stack 详细用法
  • 常用操作（竞赛必背核心接口）：
操作 功能 时间复杂度 代码示例
push(val) 将元素val压入栈顶 O(1) st.push(10);
pop() 移除栈顶元素（无返回值，易错点！） O(1) st.pop();
top() 获取栈顶元素（不移除） O(1) cout << st.top();
size() 返回栈中元素个数 O(1) cout << st.size();
empty() 判断栈是否为空（空返回true） O(1) if (st.empty()) {}
注意事项

  • pop()无返回值：严禁写int x = st.pop();，会编译报错！正确写法是先通过top()获取栈顶元素，再执行pop()；

  • 空栈操作报错：空栈状态下调用top()或pop()，会导致运行错误（RE），竞赛中必须先通过empty()判断栈非空，再进行操作；

  • 不支持迭代器：无法用范围for循环、auto迭代器遍历栈，只能通过top()+pop()逐个访问并弹出元素。

（2）priority_queue（优先队列，堆）

  • 特点：自动排序，默认大顶堆，可指定为小顶堆；底层基于vector实现，核心用于贪心、TopK等场景；

  • 常用操作及扩展说明：

操作 功能 时间复杂度 代码示例
push(val) 元素入堆，自动调整堆结构 O(log n) big_heap.push(3);
pop() 堆顶元素出堆（无返回值） O(log n) big_heap.pop();
top() 获取堆顶元素（大顶堆为最大，小顶堆为最小） O(1) cout << big_heap.top();
size() 返回堆中元素个数 O(1) cout << big_heap.size();
empty() 判断堆是否为空 O(1) if (big_heap.empty()) {}
补充说明：小顶堆声明需指定容器和比较函数，格式为 priority_queue<int, vector<int>, greater<int>> small_heap;，竞赛中高频用于求最小TopK问题。

注意事项（竞赛高频易错）

  • 默认排序规则：默认是大顶堆（降序），小顶堆必须显式声明priority_queue<int, vector<int>, greater<int>>，不可遗漏中间的vector<int>；

  • 自定义类型排序：存储自定义结构体（如pair、自定义结构体）时，需重载比较函数，否则会编译报错；

  • pop()仅删除堆顶：无法删除指定元素，若需删除非堆顶元素，需重新构建堆或换用其他容器（如set）；

  • 效率注意：push和pop均为O(log n)，适合TopK、贪心等场景，数据量极大时需注意时间限制。

4.3 关联式容器

（1）map（有序映射，红黑树）

  • 特点：键值对存储，键唯一且有序（红黑树实现），查找、插入、删除均为O(log n)；核心用于映射、统计、字符匹配等场景；

  • 常用操作及扩展操作：

操作 功能 时间复杂度 代码示例
mp[key] = val 插入/修改键值对 O(log n) mp["Alice"] = 100;
count(key) 判断键是否存在（返回0或1） O(log n) if (mp.count("Bob")) {}
find(key) 查找键，返回对应迭代器（未找到返回mp.end()） O(log n) auto it = mp.find("Alice");
erase(key) 删除指定键对应的键值对 O(log n) mp.erase("Bob");
lower_bound(key) 返回第一个≥key的键值对迭代器 O(log n) auto it = mp.lower_bound(5);
upper_bound(key) 返回第一个>key的键值对迭代器 O(log n) auto it = mp.upper_bound(5);
size() 返回键值对个数 O(1) cout << mp.size();

注意事项

  • []操作的坑：mp[key]若key不存在，会自动插入该key，值为默认值（如int为0），仅判断key是否存在时，优先用count(key)或find(key)，避免误插入；

  • 迭代器失效：删除迭代器指向的元素后，该迭代器会失效，不可继续使用，需通过it = mp.erase(it)更新迭代器；

  • 排序规则：按键升序排列，不可直接修改键值（会破坏排序结构），需删除原键值对后重新插入；

  • 效率对比：map查询、插入、删除均为O(log n)，若无需排序，优先用unordered_map（O(1)平均效率），但需注意哈希冲突。

（2）set（有序集合，红黑树）

  • 特点：元素唯一且有序（红黑树实现），查找、插入、删除均为O(log n)；核心用于去重、排序、区间查询等场景；

  • 常用操作及扩展操作：

操作 功能 时间复杂度 代码示例
insert(val) 插入元素（重复元素自动忽略） O(log n) st.insert(3);
count(val) 判断元素是否存在（返回0或1） O(log n) if (st.count(1)) {}
find(val) 查找元素，返回对应迭代器（未找到返回st.end()） O(log n) auto it = st.find(5);
erase(val/it) 删除元素（按值或迭代器，迭代器删除更快） O(log n) st.erase(3); / st.erase(it);
lower_bound(val) 返回第一个≥val的元素迭代器 O(log n) auto it = st.lower_bound(2);
upper_bound(val) 返回第一个>val的元素迭代器 O(log n) auto it = st.upper_bound(2);
size() 返回元素个数 O(1) cout << st.size();
